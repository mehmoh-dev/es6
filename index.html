ECMAScript 6 in React JS

Learn the most useful ES6 features that make React code easier to understand, write, and maintain. This page shows simple examples and clear explanations for beginners. 

import React from 'react';

What is ES6?

ECMAScript 6, also called ES2015, introduced modern JavaScript features that work especially well with React. These language improvements help you write cleaner components, manage data more safely, and avoid repeated code.

Note: ES6 is now standard in modern React development. Learning it will make React easier to use.

What is ES6 and why do we use it in React?

ES6 (ECMAScript 2015) is the modern version of JavaScript that makes React code cleaner and more readable. Most React features, like Hooks and Components, rely heavily on these features.

Why does this matter? Without ES6, React would be much more verbose (wordy). These features allow you to write complex UI logic with very few lines of code.

1. Arrow Functions

Arrow functions provide a shorter syntax for writing functions. They are used for defining components and handling events.

❌ Before ES6:

function Welcome() {
  return <h1>Hello!</h1>;
}
✅ With ES6 (React Standard):

const Welcome = () => {
  return <h1>Hello!</h1>;
};

Arrow functions make components shorter and easier to read. The => is called "arrow" syntax.

2. Destructuring

This allows you to "unpack" values from objects into distinct variables. This is the standard way to handle props in React components.

Example:

// Instead of using 'props.name' and 'props.age'
const UserProfile = ({ name, age }) => {
  return (
    <div>
      <p>Name: {name}</p>
      <p>Age: {age}</p>
    </div>
  );
};

Instead of writing props.name, you can directly use name. This makes your code cleaner and easier to read at a glance.

3. Template Literals

Template literals use backticks (`) and ${'{}'} to join strings and variables easily. This is great for dynamic CSS classes and dynamic strings.

Example:

const status = "active";
const isDisabled = false;

// Generates: <button class="btn btn-active">
<button className={`btn btn-${status} ${isDisabled ? 'disabled' : ''}`}>
  Click Me
</button>

With template literals, you can embed variables directly into strings using ${'{}'}. This makes dynamic class names and conditional styling much easier.

4. The Spread Operator (...)

The spread operator allows you to quickly copy all or part of an existing object or array into another. This is essential when updating state in React.

Example (Updating State):

const [user, setUser] = useState({ name: 'John', age: 25 });

// Update age while keeping the name unchanged
setUser({ ...user, age: 26 });

// Instead of doing this (which might delete other properties):
// setUser({ age: 26 }); // WRONG! This deletes the name

The spread operator {'{ ...user }'} copies all properties from user, then the age: 26 overwrites just the age. This is called an "immutable update" and is very important in React.

5. Map Method (.map())

In React, we use .map() to iterate through an array and generate a list of elements. This is how you display lists in React.

Example:

const products = ['Laptop', 'Phone', 'Tablet'];

const ProductList = () => (
  <ul>
    {products.map((item, index) => (
      <li key={index}>{item}</li>
    ))}
  </ul>
);

// Output:
// <ul>
//   <li>Laptop</li>
//   <li>Phone</li>
//   <li>Tablet</li>
// </ul>

.map() loops through each item in the array and creates a new element for each one. Always use a unique key (like key={item.id}) to help React keep track of list items.

    Why this code is beginner-friendly

    • Each component is small and focused.
    • Props are passed clearly to child components.
    • The state update uses an immutable pattern with spread syntax.

    What to remember

    • const is best for values that do not change.
    • Use destructuring to access props like {'{ id, text, completed }'}.
    • Spread syntax {'{ ...todo }'} copies object values safely.
    • JSX looks like HTML but uses JavaScript expressions inside curly braces.

    Beginner tips

    • Start by writing one component at a time.
    • Use console.log() when you want to inspect props or state.
    • Check your browser console for errors when JSX does not compile.
    • Remember that every component must return a single root element.
